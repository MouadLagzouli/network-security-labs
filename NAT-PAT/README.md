# TP Réseau : NAT et PAT avec Cisco Packet Tracer

## Objectif du TP

Ce TP présente plusieurs configurations de traduction d’adresses IP avec Cisco Packet Tracer.

L’objectif est de comprendre la différence entre :

- NAT statique
- NAT dynamique
- PAT avec une seule adresse publique
- PAT avec un pool d’adresses publiques

## Technologies utilisées

- Cisco Packet Tracer
- Routeurs Cisco
- Adressage IP
- NAT statique
- NAT dynamique
- PAT
- Tests de connectivité

## Fichiers du projet

| Fichier | Description |
|---|---|
| NAT STATIQUE.pkt | Configuration du NAT statique |
| NAT DYNAMIQUE.pkt | Configuration du NAT dynamique |
| PAT.pkt | Configuration du PAT |
| PAT 1 seul address publique.pkt | PAT avec une seule adresse publique |
| PAT pool adreses publiqiues.pkt | PAT avec un pool d’adresses publiques |

## Notions étudiées

### NAT statique

Le NAT statique permet d’associer une adresse IP privée à une adresse IP publique fixe.

### NAT dynamique

Le NAT dynamique permet de traduire des adresses privées vers un pool d’adresses publiques.

### PAT

Le PAT permet à plusieurs machines d’un réseau privé d’utiliser une même adresse IP publique grâce aux numéros de ports.

## Tests réalisés

- Test de connectivité entre les réseaux
- Vérification de la traduction d’adresses
- Simulation de l’accès d’un réseau privé vers un réseau externe
- Comparaison entre NAT statique, NAT dynamique et PAT

## Compétences acquises

- Comprendre le fonctionnement du NAT et du PAT
- Configurer la traduction d’adresses sur un routeur Cisco
- Utiliser Cisco Packet Tracer pour simuler un réseau
- Tester la connectivité réseau
- Analyser la différence entre NAT statique, NAT dynamique et PAT
